'spawners.csv' currently contains estimates taken directly from original Cox-Rogers spreadsheet that was compiled for 2015 recovery plan. Modifications have been made by Kate McGivney in a review in 2026. 



* format of the sheet has been adjusted to include more information (method of survey, method of estimate, reliability of estimate, notes on source of data, if data are included in Nuseds). 
* Total spawners lists the escapement estimate produced for Atnarko Sockeye that spawn above Stillwater lake and beyond. No spawners observed in Atnarko River below the lake chain are included in this dataset.
* This dataset aims to split total spawners into two 'life history' categories. Above Lonesome Lake includes spawners that are seen just above Lonesome lake and beyond.. In this dataset, these lake sections were included in the sum of spawners Above Lonesome, when numbers were provided. Above Stillwater includes spawners thought to be returning to spawn Above Stillwater lake. Though some annual estimate breakouts (holding sockeye just in the inlet of Stillwater Lake were included in totals to get portion-- only at peak of spawn). 

  * to break escapement by spawning location the following was done:

    * look at all surveys across the watershed, and identify estimated number of live and dead sockeye (or counts if estimates are not possible) by section. Sum all the sections above Lonesome Lake together for their peak observations. Use the ratio between the sockeye seen Above Lonesome and beyond with the spawners seen Above Stillwater to portion total escapement to each spawning location.  E.g. counted fish/total counted fish \* total estimate.
    * caveats: 

      * not all years surveyed beyond Tenas Lake (ie "Above Lonesome")
      * Recent work (McGivney 2022) found spawners from Above Tenas and Above Rainbow were actually River-type genetically (and may perhaps not have lake-type life history)-- however the number of these sockeye relative to those seen spawning Above Lonesome is minimal.
      * not all years provided dead sockeye so live estimates/counts were used only
      * not all years used the same counting methods/estimate methods-- 
      * the usual visual survey method caveats  (lighting, envrionmental conditions, diff observers, diff mode of counting etc.)

Information included in dataset:

* Survey mode describes how data was collected.
* estimate method describes how the estimate was created
* Residency describes the residency time if an AUC was applied
* Reliability describes the confidence in the total estimate (if it was provided)
* Flag/follow-up notes any estimate that has suspicious entries, or requires revisiting
* the original notes from Matt Mortimer and Steve Cox-Rogers have been retained, or added notes from KM. Data source column was added by KM to note anything that is different about the source in a review.
* NuSEDs SIL and SEN status notes where /if in NuSEDs.
* other KM comments: details on flags and follow- ups amongst others.
